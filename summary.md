## (3) Fundamental of UI/UX

- Pengertian User Interface (UI) adalah sebuah tampilan visual dari sebuah aplikasi digital guna untuk menampilkan informasi dan sebagai medium dari action yang dilakukan oleh user
- Pengertian User Experience (UX) adalah sebuah pengalaman atau kesan 
- UI/UX Design = sebuah proses pembuatan suatu tampilan aplikasi atau website dengan tujuan untuk memudahkan pengguna ketika menggunakan aplikasi atau website
- Perbedaan antara UI dan UX, yaitu:
    - UI meliputi:
        1. Visual Design
        2. Colors
        3. Graphic Designer
        4. Layout
        5. Typography
    - UX meliputi:
        1. Interaction Design
        2. Wireframe dan Prototypes
        3. Information Architect
        4. User Research
        5. Scenarios
- Bidang kerja UI/UX dibagi menjadi beberapa peran, antara lain:
    - UX Writer = berperan untuk mengerjakan dan memilih kata2 yang tepat pada website
    - UI Designer = berperan fokus kepada pembuatan tampilan sebuah sistem
    - UX Engineer = berperan dalam bidang prototyping dalam bentuk html/css
    - Interaction designer = berperan dalam animasi dan mengatur perpindahan transisi satu ke transisi lain selama user menggunakan sistem tersebut
    - UX Researcher = berperan dalam melalukan research, berkomunikasi dengan user, validasi dan user testing.
    - UI/UX Designer yang merupakan gabungan antara UI Designer dan UX researcher
- Design thinking adalah sebuah fundamental berupa teori-teori yang harus dipahami dan dikuasi oleh seorang UI/UX designer.
- Design thinking dibagi menjadi 5 tahapan, yaitu empathize, define, ideate, prototype, dan test.
    - Empathize adalah sebuah proses bagaimana menganalisa dan memahami perilaku user/pengguna dalam aktivitas sehari-hari ataupun kegiatan tertentu.
    - Define adalah sebuah proses bagaimana kita menganalisa data hasil proses empathize dan menemukan masalah utama dari user/pengguna
    - Ideate adalah sebuah proses pembuatan soluasi dari masalah yang sdah teridentifikasi dalam bentuk sebuah interface aplikasi atau website
    - Prototype adalah proses dimana kita membuat versi minimal yang bisa digunakan oleh user dan bertujuan untuk user testing
    - Test adalah proses dimana kita mencoba solusi interface yang sudah kita buat kepada user dengan tujuan mendapatkan data dari solusi yang kita buat
- Software yang digunakan dalam UI/UX Designer meliputi:
    - Dokumentasi dan Analisa = Aplikasi google docs dan google sheets
    - Design dan Prototyping = aplikasi sketch dan figma
    - Handover = aplikasi zeplin
    - Online testing = aplikasi maze dan hotjar
- Basic Rule bagi UI/UX Designer
    1. Memahami user, yaitu ketika kita memahami siapa user kita dan bagaimana perilaku mereka, maka solusi yang kita buat akan lebih tepat.
    2. Kejelasan sistem, dengan memberikan feedback yang jelas, user akan lebih mudah mengatasi permasalahan yang dia hadapi saat menggunakan aplikasi atau website yang kita rancang.
    3. Menyerupai dunia nyata, user akan lebih mudah memahami ketika elemen pada aplikasi atau website menyerupai dunia nyata
    4. Kebebasan pengguna, user akan lebih nyaman ketika mereka mempunyai pilihan ketika menggunakan aplikasi atau website
    5. Konsisten, dengan melakukan konsistensi dan merapkan standar ketika mendesain, kita dapat memperoleh dua keuntungan untuk pengguna yaitu mengurangi pembe;ajaran dan menghilangkan kebingungan.
    6. Mencegah error, sebagai UI/UX Designer kita juga harus meminimalisir kemungkinan kesalahan yang dilakukan user/pengguna.
    7. Mengenali daripada mengingat, elemen pada UI pastikan dapat dengan mudah dikenali oleh user daripada mengharusan mereka mengingat setiap elemen yang ada
    8. Fleksibel dan efisien, kita harus memastikan bahwa user bisa melakukan apa yang mereka inginkan secara fleksibel dan efisien
    9. Estetik dan minimalis, dengan memastikan design kita estetik dan minimalis akan meningkatkan kepuasan penggunaan dari aplikasi atau website yang kita buat

    - Low Fidelity, stage awal ketika membuat aplikasi dengan melakukan garis besar aplikasi seperti apa (sketching)

- Jenis-jenis user testing:
    1. Offline user testing, contohnya dengan menggunakan low fidelity interface
    2. Geurila user testing, yaitu testing yang dilakukan di tempat umum
    3. Online user testing, testing yang dilakukan secara online baik melalui video call atau perantara aplikasi lainnya.
